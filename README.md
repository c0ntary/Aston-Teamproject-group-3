Main

Класс Main - это точка входа в приложение.
Что делает:
    Содержит метод main() - стандартная точка запуска Java-программы
    Создает объект Menu и вызывает его метод start()
Назначение:
    Запускает консольное меню
    Инициализирует всю работу приложения

Menu

Класс Menu - это консольный интерфейс для управления приложением.
Основная логика:
1. Главное меню (start())
    Циклический показ меню
    Варианты загрузки данных: из файла, случайная, ручной ввод
    Пункт 4 - выход
2. Загрузка данных (handleMainMenu())
    Выбирает источник данных (FileData/RandomData/ManualData)
    Запрашивает количество элементов
    Загружает пользователей в список users
3. Сортировка (sortMenuLogic())
    Предлагает выбрать критерий: имя, email, ID и По четным ID
    Использует стратегии сортировки (NameSortStrategy, EmailSortStrategy, IdSortStrategy, Even_Sort_Id)
    Выводит отсортированный список
4. Подсчет вхождений (countOccurrencesMenu())
    Запрашивает ID
    Вызывает OccurrenceCounter.countOccurrences() для подсчета
5. Сохранение в файл (saveInToFile())
    Спрашивает, сохранять ли результат
    При согласии записывает в файл
6. Форматирование (formatUsers())
    Преобразует пользователей в строки формата: имя, email, id
Это главный управляющий класс, координирующий всю работу приложения.

User

Класс User - это сущность (модель данных), представляющая пользователя.
Поля: name - имя пользователя, email - электронная почта, id - идентификатор
Builder: Позволяет удобно создавать объекты: new User.builder().setName("...").setEmail("...").setId(1).build(), Реализует цепочку вызовов
Методы: Геттеры для полей, equals() и hashCode() - для сравнения объектов, toString() - для вывода информации

Data

Этот интерфейс служит контрактом для классов, которые будут реализовывать загрузку данных пользователей. Любой класс, реализующий этот интерфейс, должен предоставить конкретную реализацию метода loadData(), определяющую, откуда и как загружать пользователей (например, из базы данных, файла, внешнего API и т.д.).

FileData

Класс FileData реализует интерфейс Data для загрузки пользователей из текстового файла.
Что делает: Читает файл построчно, Парсит строки формата имя,email,id, Валидирует данные (имя только буквы, корректный email), Возвращает список из count пользователей, При ошибках возвращает пустой список
Ключевые особенности: использует стримы, автоматическое закрытие файла, встроенную валидацию данных.

ManualData

Класс ManualData реализует интерфейс Data для ручного ввода пользователей через консоль.
Что делает: Запрашивает у пользователя ввод данных для каждого пользователя, Последовательно запрашивает имя, email и ID, Валидирует каждый ввод с повторным запросом при ошибке, Возвращает список из count введенных пользователей
Валидация: Имя: только буквы (латиница/кириллица), Email: проверка формата email, ID: должно быть числом
Особенности: использует бесконечные циклы с валидацией, Scanner для чтения из консоли, автоматически повторяет запрос при некорректном вводе

RandomData

Класс RandomData реализует интерфейс Data для генерации случайных пользователей.
Что делает: Генерирует указанное количество пользователей со случайными данными, Создает count объектов User с помощью стрима
Генерация данных: Имя: случайный выбор из предопределенного списка, Email: имя в нижнем регистре + "@mail.ru и т.д.", ID: случайное число от 1,000,000,000 до 9,999,999,999
Особенности: используется Random, стримы

OutputData

Интерфейс OutputData определяет контракт для вывода данных.
Метод: void output(List<String> lines) - принимает список строк и выводит их
Назначение:
Абстрагирует способ вывода (консоль, файл, сеть и т.д.)
Позволяет легко добавлять новые реализации вывода
Используется классами ConsoleOutput и WriteInFile

ConsoleOutput

Класс ConsoleOutput реализует интерфейс OutputData для вывода данных в консоль.
Что делает:
Выводит заголовок "Отсортированный список:"
Построчно печатает переданный список строк

WriteInFile

Класс WriteInFile реализует интерфейс OutputData для записи данных в файл.
Что делает:
Записывает список строк в файл output.txt. При каждом вызове данные дописываются, а не перезаписываются. Каждая строка записывается с новой строки
Использует режим дозаписи (true параметр) - данные добавляются в конец
Автоматически закрывает файл через try-with-resources
Выводит подтверждение или сообщение об ошибке

SortStrategy

Интерфейс SortStrategy определяет контракт для стратегий сортировки.
Метод: void sort(List<User> users) - сортирует переданный список пользователей
Назначение:
Базовый интерфейс для всех алгоритмов сортировки
Позволяет реализовать паттерн Стратегия
Обеспечивает взаимозаменяемость различных способов сортировки
Реализации в проекте:
NameSortStrategy - сортировка по имени
EmailSortStrategy - сортировка по email
IdSortStrategy - сортировка по ID
Even_Sort_Id - сортировка пользователей с четными ID

SortContext

Класс SortContext - это контекст для паттерна Стратегия.
Назначение:
Управляет выбором и выполнением алгоритма сортировки
Отделяет клиентский код от конкретной реализации сортировки
Поля и методы:
    strategy - текущая стратегия сортировки
    SortContext() - пустой конструктор
    SortContext(SortStrategy initialStrategy) - конструктор с начальной стратегией
    setStrategy() - позволяет сменить стратегию во время выполнения
    executeSort() - выполняет сортировку, выбрасывает исключение если стратегия не установлена
В данном проекте этот класс используется для выбора между NameSortStrategy, EmailSortStrategy, IdSortStrategy и Even_Sort_Id.

EmailSortStrategy

Класс EmailSortStrategy реализует интерфейс SortStrategy для сортировки пользователей по email.
Что делает:
    Сортирует список пользователей по полю email
    Использует пузырьковую сортировку (bubble sort)
    Сравнивает email через compareTo() (лексикографическое сравнение строк)
Алгоритм:
    Внешний цикл: количество проходов
    Внутренний цикл: сравнение соседних элементов
    Если email текущего больше следующего - меняет их местами
Аналогичные классы: NameSortStrategy (по имени) и IdSortStrategy (по ID).

Even_Sort_Id

Это специализированная стратегия сортировки, которая сортирует только пользователей с четными ID.
Что делает:
    Отбирает пользователей с четными ID (id % 2 == 0)
    Сохраняет их индексы в оригинальном списке
    Сортирует только этих пользователей по ID (возрастание)
    Пользователи с нечетным ID остаются на своем месте

OccurrenceCounter

Класс OccurrenceCounter подсчитывает количество вхождений ID в списке пользователей.
Что делает:
    Принимает список пользователей и ID для поиска
    Создает отдельный поток для подсчета
    Использует стрим для фильтрации и подсчета
    Выводит результат


AllTests

Это тестовый класс для автоматической проверки работоспособности всех компонентов программы.
Назначение:
    Запускает набор тестов для проверки корректности работы приложения
    Собирает результаты и выводит итоговый отчет
    Помогает выявить ошибки после изменений в коде
Структура тестов:
1. Тестирование Menu (testMenu())
    Проверяет создание экземпляра меню
    Запускает меню (интерактивная часть не тестируется полностью)
    Проверяет наличие методов
2. Тестирование стратегий сортировки (testSortStrategies())
    testNameSortStrategy() - проверяет сортировку по имени
    testEmailSortStrategy() - проверяет сортировку по email
    testIdSortStrategy() - проверяет сортировку по ID
    Создает неотсортированный список и проверяет результат сортировки
3. Тестирование загрузки данных (testDataLoading())
    testFileData() - загружает из файла, проверяет количество и структуру данных
    testRandomData() - генерирует случайных пользователей, проверяет количество и валидность
    testManualData() - только создает экземпляр (т.к. требует ручного ввода)
